# Meeting Scheduling (MEET) — L1 High-Level Requirements

Zoom call scheduling within an active connection: the validated dialog and the server-side meeting-creation API.

Part of the subsystem-organized specification structure — see the [specification registry](../../README.md) for subsystem ownership, conventions, the ID scheme, and the legacy ID mapping. Detailed requirements live in [L2.md](L2.md); each L2 traces to exactly one L1 in this file.

---

## L1-MEET-010: Video call scheduling

Participants in an active connection shall schedule Zoom calls through a validated dialog backed by a server-side API that creates the meeting, persists it, and notifies the partner.
