# Security & Data Foundations (FOUND) — L1 High-Level Requirements

Cross-cutting security, authorization, and data-integrity guarantees: row-level security, server-side secrets, sanitization, schema constraints, triggers, and realtime publications.

Part of the subsystem-organized specification structure — see the [specification registry](../../README.md) for subsystem ownership, conventions, the ID scheme, and the legacy ID mapping. Detailed requirements live in [L2.md](L2.md); each L2 traces to exactly one L1 in this file.

---

## L1-FOUND-015: Security and data protection

All data access shall be protected by row-level security and server-side authorization; secrets shall remain server-side; user-provided content shall be sanitized before being rendered as HTML.

---

## L1-FOUND-016: Data integrity and persistence

The database schema shall enforce the application's status vocabularies via constraints, maintain rows via triggers, and publish realtime changes for every table the UI subscribes to.
