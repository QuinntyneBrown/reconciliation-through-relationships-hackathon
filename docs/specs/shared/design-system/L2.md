# Accessibility & Responsive Design (DSGN) — L2 Detailed Requirements

Each requirement traces to exactly one L1 in [L1.md](L1.md). Acceptance criteria are written so a test can be authored without further clarification; user-visible strings are quoted verbatim from source. Requirements state intended behavior; a `**Status:** Not satisfied` line marks requirements the running system does not yet meet (see [known gaps](../../GAPS.md)). Conventions, the ID scheme, and the acceptance-test trace-header format are defined in the [specification registry](../../README.md).

---

## L2-DSGN-073: Accessible form validation pattern

**Traces to:** L1-DSGN-017

Form validation shall follow the accessible error-summary pattern.

**Acceptance Criteria:**

1. Given a failed submit, when errors render, then an `alert`-role error summary receives focus and lists one anchor link per error.
2. Given an invalid field, when it renders, then it carries `aria-invalid="true"` and `aria-describedby` pointing at its error message; activating the summary link focuses the field.
3. Given a corrected field, when its value becomes valid, then both the inline error and its summary entry are removed without a fresh submit.

## L2-DSGN-074: Text contrast

**Traces to:** L1-DSGN-017

Primary call-to-action text shall meet WCAG AA contrast.

**Acceptance Criteria:**

1. Given the landing page CTAs ("Begin your journey", "Find your place in the circle"), when computed foreground/background contrast is measured, then the ratio is at least 4.5:1.

## L2-DSGN-075: Mobile navigation

**Traces to:** L1-DSGN-018

Narrow viewports shall use a hamburger-triggered navigation sheet with the same destinations and current-page marking.

**Acceptance Criteria:**

1. Given a 390px-wide viewport, when a signed-in participant opens the mobile navigation, then it lists the role's destinations, marks the current page with `aria-current="page"`, and navigating (e.g., "Regional map" → `/map`) works.

---

## L2-DSGN-076: Cross-viewport layout

**Traces to:** L1-DSGN-018

Every page shall remain usable from extra-small through extra-large viewports.

**Acceptance Criteria:**

1. Given viewport widths of 320px and 1280px, when the landing page renders, then key content (hero, tagline, CTAs) is fully visible with no horizontal page scrolling.
2. Given viewport widths from <576px (XS) through ≥1200px (XL), when any authenticated page renders, then its primary actions remain reachable (via the mobile navigation on narrow screens per L2-DSGN-075).
