# Authentication & Access (AUTH) — L1 High-Level Requirements

Account creation, sign-in, session establishment, and the journey- and role-based gating that controls which routes a user can reach.

Part of the subsystem-organized specification structure — see the [specification registry](../README.md) for subsystem ownership, conventions, the ID scheme, and the legacy ID mapping. Detailed requirements live in [L2.md](L2.md); each L2 traces to exactly one L1 in this file.

---

## L1-AUTH-002: Authentication and session management

Users shall create accounts and sign in with email and password; sessions shall be established via the auth callback exchange and shall be terminable via sign-out.

---

## L1-AUTH-003: Journey and role-based access control

Every route shall be gated by authentication state, role (participant or facilitator), and journey stage (`onboarding_completed`, `learning_completed`), so users can only reach the stage of the journey they have unlocked.
