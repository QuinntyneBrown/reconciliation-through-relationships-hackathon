# Guided Onboarding & Intake (ONBRD) — L2 Detailed Requirements

Each requirement traces to exactly one L1 in [L1.md](L1.md). Acceptance criteria are written so a test can be authored without further clarification; user-visible strings are quoted verbatim from source. Requirements state intended behavior; a `**Status:** Not satisfied` line marks requirements the running system does not yet meet (see [known gaps](../GAPS.md)). Conventions, the ID scheme, and the acceptance-test trace-header format are defined in the [specification registry](../README.md).

---

## L2-ONBRD-015: Five-step intake wizard with persistence

**Traces to:** L1-ONBRD-005

Onboarding shall guide the participant through five steps — Basic Info, Location & Treaty, Faith & Interests, Availability & Format, Matching Preferences — persist the profile, and open the learning journey.

**Acceptance Criteria:**

1. Given a participant on `/onboarding`, when each step renders, then a progress indicator shows "Step N of 5" for the current step.
2. Given completed earlier steps, when the participant navigates Back, then previously entered values (text fields, checked options) are preserved.
3. Given all steps valid, when the participant activates "Complete profile", then the `profiles` row is updated with every collected field, `onboarding_completed` is set to `true`, the toast "Profile saved! Let's start your learning journey." appears, and the browser navigates to `/learn`.
4. Given interests entered as tags, when they are added (via Enter or the "Add" button), then they are stored lowercased and de-duplicated, and each tag can be removed before submission.
5. Given no authenticated session at submit time, when the participant submits, then the toast "Session expired. Please sign in again." appears and the browser navigates to `/auth/login`.

## L2-ONBRD-016: Step 1 validation with error summary

**Traces to:** L1-ONBRD-005

Step 1 (Basic Info) shall validate on Continue and present a focusable error summary with anchor links plus inline field errors, using exact messages.

**Acceptance Criteria:**

1. Given an empty step 1, when the participant activates Continue, then an alert "Please fix 6 errors to continue" receives focus and lists the linked messages: "Enter your first name.", "Enter your last name.", "Enter an age from 13 to 120.", "Choose whether you are Indigenous.", "Choose your sex.", and "Choose at least one required participation category."
2. Given the error summary, when the participant activates a summary link, then focus moves to the corresponding field.
3. Given at least one participation category selected, when none of it is `indigenous_leader`, `indigenous_individual`, or `non_indigenous_individual`, then the category error is still shown (only those three satisfy the requirement).
4. Given a corrected field, when its value becomes valid, then that error disappears from the summary and the inline message without a new submit.

## L2-ONBRD-017: Age validation

**Traces to:** L1-ONBRD-005

The age field shall accept only integers from 13 to 120.

**Acceptance Criteria:**

1. Given an age of 12, when the participant activates Continue on step 1, then the alert "Please fix 1 error to continue" appears with the message "Enter an age from 13 to 120." and the field carries `aria-invalid="true"`.
2. Given an age of 13, 120, or any integer between, when step 1 is otherwise valid, then no age error is shown.

## L2-ONBRD-018: Later-step required-group validation

**Traces to:** L1-ONBRD-005

Steps 2–4 shall validate their required groups on Continue with exact messages (step 5 has no required fields).

**Acceptance Criteria:**

1. Given an empty step 2, when the participant activates Continue, then "Please fix 2 errors to continue" appears with "Enter your city, town, or county." and "Choose a province or territory." shown in both the summary and inline.
2. Given an empty step 3, when the participant activates Continue, then the message "Choose a faith tradition or prefer not to say." appears.
3. Given an empty step 4, when the participant activates Continue, then two errors appear: "Choose at least one participation format." and "Choose at least one language."
4. Given step 2's province selector, when it is opened, then it lists full province and territory names (e.g., "Saskatchewan", never two-letter codes).

## L2-ONBRD-019: Failed profile save is retryable

**Traces to:** L1-ONBRD-005

A failed final save shall keep the participant on the last step with a retryable error.

**Acceptance Criteria:**

1. Given the `profiles` update fails, when the participant activates "Complete profile", then the toast "Something went wrong. Please try again." appears, the wizard remains on the Matching Preferences step, and the "Complete profile" button is re-enabled.
2. Given the failure, when the participant retries and the write succeeds, then the success flow of L2-ONBRD-015 proceeds normally.

## L2-ONBRD-061: Intake API listing

**Traces to:** L1-ONBRD-014

`GET /api/participants` shall return the participant collection from the configured repository.

**Acceptance Criteria:**

1. Given the endpoint, when it is called, then it responds `200` with `{"participants": [...]}` from the repository selected by `DATA_SOURCE`.

## L2-ONBRD-062: Intake API validation

**Traces to:** L1-ONBRD-014

`POST /api/participants` shall validate the payload against the shared intake schema and reject invalid submissions with field-level errors.

**Acceptance Criteria:**

1. Given an invalid payload, when the endpoint is called, then it responds `400` with `{"error":"Validation failed"}` and flattened field issues using the schema's messages (e.g., "Name is required", "Enter a valid email", "You must be at least 13 years old", "Select at least one participation category", "City, town, or county is required", "Add at least one language").
2. Given a valid payload, when the endpoint is called, then it responds `201` with `{"participant": {...}}` for the created participant.
