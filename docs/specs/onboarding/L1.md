# Guided Onboarding & Intake (ONBRD) — L1 High-Level Requirements

The validated five-step participant intake wizard, and the schema-validated participant intake API seam.

Part of the capability-based specification structure — see the [capability registry](../README.md) for conventions, the ID scheme, and the legacy ID mapping. Detailed requirements live in [L2.md](L2.md); each L2 traces to exactly one L1 in this file.

---

## L1-ONBRD-005: Onboarding intake

Participants shall complete a validated five-step intake wizard that persists their profile to the `profiles` table and unlocks the learning journey.

---

## L1-ONBRD-014: Participant intake API

The system shall expose a schema-validated `GET`/`POST /api/participants` endpoint backed by the repository seam. (Reference seam: not used by the live onboarding UI.)
