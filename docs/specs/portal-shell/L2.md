# Portal Shell & Navigation (SHELL) — L2 Detailed Requirements

Each requirement traces to exactly one L1 in [L1.md](L1.md). Acceptance criteria are written so a test can be authored without further clarification; user-visible strings are quoted verbatim from source. Requirements state intended behavior; a `**Status:** Not satisfied` line marks requirements the running system does not yet meet (see [known gaps](../GAPS.md)). Conventions, the ID scheme, and the acceptance-test trace-header format are defined in the [capability registry](../README.md).

---

## L2-SHELL-013: Primary navigation with current-page indication

**Traces to:** L1-SHELL-004

Authenticated pages shall present a primary navigation with the destinations for the user's role and mark the current page.

**Acceptance Criteria:**

1. Given a signed-in participant on `/dashboard`, when the navigation renders, then it offers Home (`/dashboard`), Learning (`/learn`), Connections (`/connections`), and Regional map (`/map`), and the current item carries `aria-current="page"`.
2. Given a signed-in facilitator, when the navigation renders, then it offers Overview, Participants, Matches, Cohorts (`/map`), and Settings, with the role label "Facilitator".
3. Given any navigation item, when it is activated, then the browser navigates to that destination.

## L2-SHELL-014: Account menu

**Traces to:** L1-SHELL-004

Every authenticated area shall offer an account menu with a profile shortcut (participants) and sign-out.

**Acceptance Criteria:**

1. Given a signed-in participant, when they open the account menu, then a "My profile" item navigates to `/profile/{their-id}`.
2. Given a user on `/onboarding`, `/learn`, `/dashboard`, or `/facilitator`, when they choose Sign out, then the session ends and the browser navigates to `/auth/login` (from onboarding/learning) or `/` (from dashboard/facilitator), after which protected routes redirect to `/auth/login`.
