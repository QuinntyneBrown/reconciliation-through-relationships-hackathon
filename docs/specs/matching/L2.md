# Matching Engine & Discovery (MATENG) — L2 Detailed Requirements

Each requirement traces to exactly one L1 in [L1.md](L1.md). Acceptance criteria are written so a test can be authored without further clarification; user-visible strings are quoted verbatim from source. Requirements state intended behavior; a `**Status:** Not satisfied` line marks requirements the running system does not yet meet (see [known gaps](../GAPS.md)). Conventions, the ID scheme, and the acceptance-test trace-header format are defined in the [specification registry](../README.md).

## L2-MATENG-024: Matching algorithm

**Traces to:** L1-MATENG-007

Match recommendations shall pair only Indigenous with non-Indigenous participants and score each pair out of 100 using fixed weights.

**Acceptance Criteria:**

1. Given a participant, when candidates are scored, then only candidates whose `is_indigenous` differs from the participant's are considered.
2. Given a candidate pair, when the score is computed, then it is the sum of: location (same city case-insensitively = 30, else same province = 15, else 0), availability overlap (up to 20, from day and time overlap ratios ×10 each, rounded), shared interests (shared ÷ smaller list × 20, rounded), shared language (any = 10), same faith tradition (10), and shared participation format (any = 10).
3. Given scored candidates, when the dashboard selects recommendations, then it keeps candidates that are in an approved match or score at least 40, sorted descending, and shows at most 5.

## L2-MATENG-025: Recommendation cards

**Traces to:** L1-MATENG-007

Each recommendation shall show the candidate, the rounded percentage score, the positive criteria, and a state-appropriate action.

**Acceptance Criteria:**

1. Given a recommendation, when the card renders, then it shows the candidate's name, city/province, Indigenous or Non-Indigenous badge, the score as a rounded percentage (e.g., "83%"), chips only for criteria that scored points, and a "View profile" link to `/profile/{id}`.
2. Given an active connection with the candidate, when the card renders, then the action is "Open chat" linking to `/connections/{id}`; given a connection under review, the action reads "Pending" with the tooltip "Waiting for a facilitator to approve this match"; given no connection, the action routes to the candidate's profile.

## L2-MATENG-026: Waitlist and empty recommendation states

**Traces to:** L1-MATENG-007

Non-Indigenous participants with no available match shall see a waitlist explanation; an empty recommendation list shall be explained.

**Acceptance Criteria:**

1. Given a non-Indigenous participant with zero recommendations, when the dashboard renders, then an informational alert beginning "You're on the waitlist" explains they will be notified when an Indigenous participant is matched with them.
2. Given zero recommendations, when the Recommended tab renders, then the empty state "No recommendations yet" with "A facilitator is reviewing potential matches for you. Check back soon." is shown.

## L2-MATENG-027: Cohort-forming banner

**Traces to:** L1-MATENG-007

Elected leaders shall be told when enough same-city participants exist to form a cohort. (Display only — see GAP-009 for the inactive "Create cohort" action.)

**Acceptance Criteria:**

1. Given a participant whose `participation_categories` includes `elected_leader`, who belongs to no cohort, and whose city has at least 5 eligible participants (including themselves), when the dashboard renders, then a banner announces "A reconciliation cohort is forming in {city}!" with the eligible count.
2. Given any of those conditions false, when the dashboard renders, then no cohort banner appears.

## L2-MATENG-028: Participant directory search and filters

**Traces to:** L1-MATENG-007

The "All participants" tab shall list eligible participants with search, filters, and pagination.

**Acceptance Criteria:**

1. Given the directory, when it loads, then it lists only participants with `onboarding_completed = true`, `learning_completed = true`, and role `participant`, excluding the viewer, with a result count (e.g., "3 participants").
2. Given a search term, when it matches a name, city, or interest, then only matching participants remain and the count updates.
3. Given province (full names), background (Indigenous/Non-Indigenous), and faith filters, when combined, then results satisfy all active filters.
4. Given no matching participants, when filters are applied, then the empty state "No participants match your filters." is shown.
5. Given more than 9 results, when the list renders, then it paginates at 9 per page with a "Showing X–Y of N" indicator.

## L2-MATENG-029: Directory map view honors consent

**Traces to:** L1-MATENG-007

The directory's map view shall plot only consenting participants at city level.

**Acceptance Criteria:**

1. Given the directory, when the participant switches to Map view, then only participants with `map_consent = true` and coordinates are plotted, with the caption "Only showing participants who consented to map display. Locations are city-level only."
2. Given no consenting participants in the filtered set, when the map renders, then it shows "No participants have consented to map display in this view."
3. Given the map view, when the participant switches back to List, then the list view returns.

## L2-MATENG-030: Dashboard connections tab

**Traces to:** L1-MATENG-007

The Connections tab shall summarize the participant's connections with status badges and filtering.

**Acceptance Criteria:**

1. Given connections, when the tab renders, then each row shows the partner, their city, and a status badge: "Active"; pending with the other party's action outstanding ("Waiting for {name} to accept" / "{name} is waiting for you to accept"); or under review (tooltip "Waiting for a facilitator to approve this match").
2. Given no connections, when the tab renders, then "No active connections yet" appears with an "Explore recommendations" action.
3. Given more than 10 connections, when the tab renders, then it paginates at 10 per page, with an all/active/pending/under-review status filter.
