# Learning Journey (LEARN) — L2 Detailed Requirements

Each requirement traces to exactly one L1 in [L1.md](L1.md). Acceptance criteria are written so a test can be authored without further clarification; user-visible strings are quoted verbatim from source. Requirements state intended behavior; a `**Status:** Not satisfied` line marks requirements the running system does not yet meet (see [known gaps](../GAPS.md)). Conventions, the ID scheme, and the acceptance-test trace-header format are defined in the [specification registry](../README.md).

---

## L2-LEARN-020: Audience-filtered learning modules

**Traces to:** L1-LEARN-006

`/learn` shall present only modules whose audience matches the participant (`indigenous` or `non_indigenous` per `is_indigenous`, plus `all`), ordered by `order_index`, with required and optional modules distinguished.

**Acceptance Criteria:**

1. Given a non-Indigenous participant, when `/learn` loads, then only modules with audience `non_indigenous` or `all` are listed, in `order_index` order.
2. Given an Indigenous participant, when `/learn` loads, then only modules with audience `indigenous` or `all` are listed.
3. Given a module with `is_required = false`, when the sidebar renders, then it carries an "Optional" badge, and overall progress counts required modules only ("X/Y complete").

## L2-LEARN-021: Module progression

**Traces to:** L1-LEARN-006

Participants shall open any listed module and mark it complete, recording progress and advancing to the next incomplete module.

**Acceptance Criteria:**

1. Given the module list, when the participant selects a different module, then the content area swaps to that module (video modules render an embedded frame from `content_url`; text modules render `content_body` as formatted text).
2. Given an open module, when the participant activates "Mark complete", then a `learning_progress` row is upserted with `completed = true` for that user and module, the toast "Module complete!" appears, and the view advances to the next incomplete module.

---

## L2-LEARN-022: Learning journey completion

**Traces to:** L1-LEARN-006

Completing all required modules shall mark learning complete and move the participant to the dashboard.

**Acceptance Criteria:**

1. Given the last incomplete required module, when the participant marks it complete, then `profiles.learning_completed` is set to `true`, the toast "Learning journey complete! Taking you to your dashboard." appears, and the browser navigates to `/dashboard`.
2. Given a participant with `learning_completed = true`, when they visit `/learn` directly, then the modules remain reviewable and the account menu offers "Dashboard".

---

## L2-LEARN-023: Failed progress save is retryable

**Traces to:** L1-LEARN-006

A failed progress write shall leave the module incomplete and retryable.

**Acceptance Criteria:**

1. Given the `learning_progress` write fails, when the participant activates "Mark complete", then the toast "Failed to save progress. Please try again." appears and the "Mark complete" button is re-enabled.
