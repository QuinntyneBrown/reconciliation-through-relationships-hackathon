Toast catalog

Every toast the app can raise, grouped by feature, with the exact message text and its source file and line.

Toasts are rendered by sonner configured in src/components/ui/sonner.tsx: a dark inverse surface (--inverse), semibold body text, and a per-type lucide icon tinted inverse-accent (ochre). Success uses CircleCheck, error uses OctagonX, and info uses Info. Message strings are copied verbatim; where a message is interpolated or dynamic, the interpolation is shown and annotated.

Realtime bell

Notification toasts · 1 generic shape

Every notification row inserted for the current user also raises a live toast through the realtime subscription in src/components/notification-center.tsx. This uses the default (untyped) toast(title, { description, action }) shape — no status icon — with the notification’s title, its body as the description, and a “View” action button that routes to the related connection or dashboard.
