Section 1

Validation

Validation runs in two places. The onboarding wizard validates each step client-side and shows inline errors — an error summary at the top of the step plus a red message under each bad field. The shared zod schema in src/domain/schema.ts validates the /api/participants route server-side. Both layers are catalogued below.

Onboarding wizard · client-side

Onboarding inline validation

Each onboarding step validates on submit and blocks navigation until fixed, using src/app/onboarding/validation.ts for the messages and OnboardingErrors.tsx for the rendering. Two things appear: a destructive error summary at the top of the step with anchor links to each bad field, and a red per-field message below the input. The zod messages further down are the API-side counterpart — the same intake validated server-side at /api/participants.

Signup client-side rules

Password rules fire as toasts, not field errors

The sign-up form checks the password client-side before calling Supabase. On this form both rules surface as toast.error (see the toast catalog) rather than inline field messages — unlike the onboarding wizard above, which renders inline field errors.

Required-field convention

Required labels append a literal “ *”

In the onboarding wizard, required fields append a trailing asterisk to the label text (e.g. <Label>First name *</Label>). Optional fields (Bio, Additional matching information) carry no asterisk.

API errors

JSON error responses from the two API routes. Each returns { error: "…" } at the given HTTP status; the Zoom route adds a detail / status field on failures, and the participants route adds issues (the flattened zod error).

RouteStatuserror stringWhenSource
POST /api/zoom/create-meeting 401 Unauthorized No authenticated user. route.ts:51
400 Missing required fields Any of connectionId / scheduledAt / durationMinutes / topic is missing. route.ts:58
403 Forbidden Caller is not a participant in the connection. route.ts:71
502 Failed to create Zoom meeting Zoom’s meetings API returned a non-OK response. route.ts:101
500 Failed to save meeting Meeting row insert failed after Zoom created the meeting. route.ts:124
500 Zoom integration error Caught exception — e.g. missing Zoom S2S OAuth env vars. route.ts:144
POST /api/participants 400 Validation failed The body fails the shared zod schema; issues holds the flattened errors. route.ts:21
